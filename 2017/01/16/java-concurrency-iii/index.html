synchronized 关键字

　　先来总结一下使用Lock和Condition对象的关键：

  • 锁用来保护代码片段，任何时刻只有一个线程执行被保护的代码
  • 锁可以拥有一个或多个条件对象
  • 每个条件对象管理进入被保护的代码片段，但还不能运行的线程

　　Lock和Condition接口提供了高度的锁定控制，Java中的每个对象都有一个内部锁，如果方法用synchronized关键字生命，那么对象所将保护整个方法，也就是说要调用该方法，线程必须获得内部对象锁。
